# First Page Budget Dashboard

## 职责

首屏模块负责展示可自由支配金额、三色收入分配条和各用途预算卡片；资金用途分为日常类、固定开销类、储蓄类，只有日常类点击后会打开快速记账弹窗并扣减余额。

## 优先查看

- `index.html`: 页面结构、设置弹窗、快速记账弹窗和底部导航占位。
- `styles.css`: 白底、蓝色、淡绿色为主的 iOS 竖屏样式，包含分类分组、总览三色条、卡片底部进度条、弹窗和底部导航。
- `app.js`: `localStorage` v2 状态、三类资金模型、比例/金额双输入换算、设置保存、日常类快速记账扣减；固定/储蓄类点击保持静默。
- `manifest.webmanifest` 和 `sw.js`: PWA 安装信息与基础离线缓存。
- `scripts/dev-server.mjs`: 本地开发静态服务器，负责返回正确 MIME，避免模块脚本被浏览器拒绝加载。

## 状态说明

当前数据只保存在浏览器本地 `localStorage`，默认收入为 10000，默认用途为生活（日常类，50%）、房租（固定开销类，3000 元）、储蓄（储蓄类，20%）；`app.js` 会读取旧版 `moneyapp.budget.v1` 并规范化为 v2 结构。

总览金额只显示日常类剩余额度；总览进度条按固定开销类（黄色）、储蓄类（蓝色）、日常类（绿色）顺序展示占收入比例。固定开销类在展示层直接视为已扣除，卡片只展示开销金额且没有进度条；储蓄类视为留存，不进入记账弹窗。若后续接入多月份账本或同步存储，优先从 `app.js` 的 `categories`、`getSummarySegments()`、`toViewFund()` 和 `saveSettings()` 扩展。
